Fala pessoal, tudo bom?

Alguns dias atrás, publiquei um post falando sobre o que é mobile engagement e comentei que posteriormente falaria sobre uma ferramenta chamada Azure Mobile Engagement. Se você não leu o post que introduz o conceito, recomendo a leitura antes de continuar.

No post de hoje, começarei uma série que falará sobre o que é o Azure Mobile Engagement e como ele facilita a implementação de algumas técnicas que visam o engajamento de usuários.

Veja abaixo uma breve introdução e o que podemos fazer com essa ferramenta.

Introdução

Em 2014, a Microsoft anuncia a compra de uma empresa chamada Caaptain. A Caaptain era uma empresa francesa que oferecia alguns serviços em nuvem como push e analytics em tempo real.

Após a compra, a Microsoft integrou parte dos serviços oferecidos pela Caaptain no Azure, mas manteve o portal da antiga empresa que está em uso até hoje (07/16). Embora o portal tenha sido mantido, toda a stack foi renomeada para Azure Mobile Engagement (AME).

O AME é um SaaS oferecido pelo Microsoft Azure que possibilita uma fácil implementação de recursos de engagemento de usuários como push notification, in-app message, analytics, etc.

A configuração de envio de push e in-app message podem ser feitos pelo portal e não necessitam código. Isso possibilita que pessoas não técnicas possam trabalhar criando campanhas de engagemento. Um bom exemplo de uma área que pode tirar proveito disso é o marketing.

Com alguns cliques é possível configurar o envio de push notification segmentado, como por exemplo, para usuários que não acessam o aplicativo há 1 semana. Ou também, fazer com que uma mensagem apareça no app informando sobre uma feature que nunca foi usada por aquele usuário ou uma promoção relâmpago.

Através do portal do AME, é possível obter informações em tempo real do comportamento dos usuários. Saiba aonde os usuários estão e o que estão fazendo utilizando os recursos de event e activities oferecidos pelo sdk da ferramenta.

 

Integre com seu software de gestão

Caso você não queira usar a interface web do AME, é possível integrá-lo ao seu software.

Através de APIs rest ou do sdk para .net, é possível criar campanhas e enviar push notifications. Imagine ter esse controle dentro do seu crm ou cms?

 

Principais recursos (na minha visão)

  • Real time analytics: É possível ver em tempo real os usuários ativos no app e em qual tela eles estão.
  • Event track: Track de eventos em tempo real.
  • User path: Esse gráfico mostra uma arvore de navegação. Para quais tela o usuários vai quando está na tela x ou y.
  • Technicals: Informações sobre fabricante, versão dos s.o, rede de dados usada, etc. referente ao dispositivo.
  • Push notification: Com alguns cliques configure um serviço de push notification.
  • In-app message: Exibição de alertas em texto/html.
  • Segmentos de usuários: Crie segmentos de usuários e envie push e campanhas para usuários específicos. Na minha opinião, esse é uma das principais features.
  • Pesquisas: Crie formulários de pesquisas que serão exibidos no app.
  • Estatísticas de campanhas e pesquisas: Visualize o resultado de pesquisas e campanhas através de gráficos ou integre com o PowerBI.

Tem muitos outros recursos, mas esses são os que mais estou usando no momento. Nos próximos posts, vou mostrar todos os itens acima na prática.

 

Conclusão

Chegamos em um momento na tecnologia em que podemos ter serviços configurados com apenas alguns clicks. Serviços que antes levávamos horas, ou até dias, para implementar.

A facilidade de implementação e configuração do Azure Mobile Engagement é imensa. Os dados que podem ser obtidos através dos gráficos de comportamento de usuário podem se transformar em informações que talvez seja o diferencial que possibilite sair na frente dos concorrentes.
